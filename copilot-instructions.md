# Copilot Project Instructions

这些规则是本项目内的硬性要求，所有代码改动必须遵守。

## 必须遵守

1. 所有新增或修改的业务逻辑代码，必须添加中文注释，且注释放在对应逻辑上方。
2. 没有注释的改动视为无效改动。
3. 只允许最小改动：只改用户明确要求的范围，不改无关代码。
4. 禁止擅自“优化”“重构”或新增扩展逻辑，除非用户明确提出。
5. 修改前先阅读当前文件最新内容，避免覆盖用户刚改过的代码。
6. 修改后必须进行基础检查，确认无新增语法或编译错误。

## 沟通要求

1. 先回答改动点，再解释原因，不要长篇发挥。
2. 如需多步修改，按用户要求一项一项进行，每一步都先确认目标。
3. 用户要求“还原”时，必须立即回退本次改动范围。

## 接口改动约定（当前项目）

1. 后端返回格式调整时，优先在公共请求层做最小适配。
2. 非必要不改页面层业务逻辑。
3. 不要添加与当前需求无关的兜底逻辑或兼容分支。
